## TL;DR
Design agent with goals, tools, safety guardrails.

## Inputs
- `agent_goal` (what should it do?)
- `allowed_tools` (APIs, functions)
- `safety_requirements` (approval needed?)

## Output Contract
1. System instructions
2. Action policy
3. Failure handling
4. Safety guardrails (turn/token/time limits)

## Example
```yaml
agent_name: support-bot
goal: Resolve tickets via knowledge base
allowed_tools: [search_kb, update_ticket]
forbidden: [delete_data, refund]
max_turns: 10
token_budget: 30000
time_limit_ms: 120000
```
