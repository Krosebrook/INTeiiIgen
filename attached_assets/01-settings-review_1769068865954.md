# Settings Security Review

## TL;DR
Audit configuration for misconfig risks.

## Inputs
- `settings_structure` (schema/UI)
- `user_roles` (admin/user/guest)

## Output Contract
1. Misconfiguration risks (prioritized)
2. Safer defaults
3. Required confirmations
4. Logging recommendations

## High Risk Example
Issue: API key visible to all users
Risk: Leaked key = full access
Fix: Show last 4 chars only, require re-auth
