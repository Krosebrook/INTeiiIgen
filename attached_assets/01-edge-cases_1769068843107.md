# Edge Cases — Failure Scenarios

## TL;DR
Enumerate edge cases to ensure robust testing.

## Inputs
- `feature_description` OR `function_signature`

## Output Contract
1. Edge cases (user/system/network)
2. Risk levels (low/medium/high)
3. Top 10 "likely in prod"
4. Suggested tests

## Example
High: Empty input → Validation error
Medium: Invalid email → Client-side block
Low: Unicode in name → Handle gracefully
