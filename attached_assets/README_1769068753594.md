## ğŸš¦ When to Use Each Phase

### Phase 1: Planning (START HERE)
**Trigger:** New project, new feature, or unclear requirements  
**Output:** Clear requirements, architecture diagram, effort estimate  
**Next:** Phase 2 (Design) OR Phase 3 (Development)

### Phase 2: Design (Visual-First Projects)
**Trigger:** Building user-facing pages, templates, or flows  
**Output:** Page structures, component hierarchies, wireframes  
**Next:** Phase 3 (Development)

### Phase 3: Development (Code Generation)
**Trigger:** Writing new code, refactoring, debugging, optimizing  
**Output:** Production-ready code, tests, documentation  
**Next:** Phase 4 (Quality)

### Phase 4: Quality (Before Ship)
**Trigger:** Code written, needs validation  
**Output:** Test suite, edge cases covered, regression checks  
**Next:** Phase 5 (Deployment)

### Phase 5: Deployment (Ship Time)
**Trigger:** Code tested, ready to deploy  
**Output:** Deployed code, smoke tests passed, monitoring active  
**Next:** Phase 6 (Operations)

### Phase 6: Operations (Post-Deploy)
**Trigger:** Code in production, needs monitoring  
**Output:** Dashboards, alerts, incident responses  
**Next:** Phase 1 (next feature) OR Phase 3 (bug fixes)

### Phase 7: Agents (Automation Layer)
**Trigger:** Repetitive tasks, need automation  
**Output:** Configured agents, debugged workflows, loop prevention  
**Next:** Phase 4 (test agents) OR Phase 6 (monitor agents)

### Phase 8: Security (Continuous)
**Trigger:** ALWAYS (parallel to all phases)  
**Output:** Security reviews, hardened configs, audit logs  
**Next:** Ongoing (revisit quarterly)
